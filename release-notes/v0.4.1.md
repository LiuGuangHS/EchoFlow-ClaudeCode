# EchoFlow Code v0.4.1

这是 EchoFlow Code 的 `0.4.1` 版本，重点收敛桌面端 Electron 自动更新链路，恢复 Windows sidecar 兼容性，并将桌面端安装的全局 CLI 命令切换为独立的 `echoflow-code`。

## Highlights

- **独立正式版本线**：EchoFlow Code 使用自己的 SemVer 版本号，不再用 `rc` 后缀规避上游版本号。
- **国内更新源默认可用**：Electron 更新默认继续优先使用 `gh-proxy.org`，并保留 GitHub Releases 原始源作为检查阶段 fallback。
- **Windows 兼容性恢复**：Windows x64 sidecar 默认恢复为 `bun-windows-x64-baseline`，降低老 CPU 机器启动 sidecar 时出现 `Illegal Instruction` 的风险。
- **独立全局命令**：桌面端安装到用户 PATH 的命令统一为 `echoflow-code`。

## Fixes

- 修复 release tooling 对 `0.4.1-rc.1` 这类版本不完整支持导致的后续 bump 风险。
- 简化 updater 状态机，避免自动更新服务在初始化阶段因 feed 设置失败影响应用行为。
- 恢复 Windows baseline sidecar runtime，提高打包桌面端在旧 Windows x64 设备上的兼容性。

## Notes

- 本版本仍可能因为未签名触发 Windows SmartScreen 提示，选择“更多信息”→“仍要运行”即可继续安装。
- 已安装旧测试版的用户可直接安装本版本覆盖升级。

## 安装

到本页面底部的 Assets 里，按系统和 CPU 架构下载对应文件：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon（M 系列） | `EchoFlow-Code-0.4.1-mac-arm64.dmg` |
| macOS Intel | `EchoFlow-Code-0.4.1-mac-x64.dmg` |
| Windows x64 | `EchoFlow-Code-0.4.1-win-x64.exe` |
| Linux x64 | `EchoFlow-Code-0.4.1-linux-x86_64.AppImage` 或 `EchoFlow-Code-0.4.1-linux-amd64.deb` |
| Linux ARM64 | `EchoFlow-Code-0.4.1-linux-arm64.AppImage` 或 `EchoFlow-Code-0.4.1-linux-arm64.deb` |
