# EchoFlow-ClaudeCode v0.4.1-rc.1

这是 EchoFlow-ClaudeCode 的 `0.4.1-rc.1` 预发布版本，重点修复桌面端 Electron 自动更新链路，并调整 Windows 默认 sidecar runtime 为标准 x64 构建。

## Highlights

- **应用更新源更稳**：Windows / Electron 更新默认继续优先使用 `gh-proxy.org`，当代理源检查或下载失败时自动降级到 GitHub Releases 原始源。
- **修复同版本误更新**：当远端版本等于或低于当前版本时，不再显示为可下载更新，避免出现“当前版本和可更新版本相同但仍可点击更新”的状态。
- **Windows 默认使用标准 x64 runtime**：默认 Windows x64 包改用标准 `bun-windows-x64` sidecar runtime，优先保证主流机器性能和本地/CI 构建稳定性。

## Fixes

- 修复 Electron updater 在代理源失败时没有备用源的问题。
- 修复下载阶段代理源失败后无法重新检查并切换下载源的问题。
- 修复同版本/旧版本 metadata 被前端视为可更新版本的问题。
- 修复 Windows sidecar 默认构建目标仍使用 baseline runtime 导致本地 Bun 1.3.14 baseline compile 不稳定的问题。

## Notes

- Windows x64 默认包现在要求较新的 x64 CPU，建议 Intel 4 代 Core / AMD Excavator 或更新 CPU 使用。
- 自动更新默认升级到标准 Windows x64 包，不再为老 CPU 单独维护 baseline 自动更新通道。
- 本版本仍可能因为未签名触发 Windows SmartScreen 提示，选择“更多信息”→“仍要运行”即可继续安装。

## 安装

到本页面底部的 Assets 里，按系统和 CPU 架构下载对应文件：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon（M 系列） | `EchoFlow-ClaudeCode-0.4.1-rc.1-mac-arm64.dmg` |
| macOS Intel | `EchoFlow-ClaudeCode-0.4.1-rc.1-mac-x64.dmg` |
| Windows x64 | `EchoFlow-ClaudeCode-0.4.1-rc.1-win-x64.exe` |
| Linux x64 | `EchoFlow-ClaudeCode-0.4.1-rc.1-linux-x86_64.AppImage` 或 `EchoFlow-ClaudeCode-0.4.1-rc.1-linux-amd64.deb` |
| Linux ARM64 | `EchoFlow-ClaudeCode-0.4.1-rc.1-linux-arm64.AppImage` 或 `EchoFlow-ClaudeCode-0.4.1-rc.1-linux-arm64.deb` |
